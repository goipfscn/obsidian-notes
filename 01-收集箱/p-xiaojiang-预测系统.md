# P小将 风格预测系统

## 目标
预测 P小将 的买入喜好，他们买入后卖出获利

## 核心逻辑
```
新币特征 → 匹配 P小将 画像 → 提前买入 → P小将买入确认 → 卖出
```

## 系统架构

```
数据层:
- 链上数据：买入记录、持仓变化、时间戳
- Twitter数据：发推内容、互动内容

特征工程层:
- 市值：5位/6位/7位+
- 叙事：AI/meme/链游/VC/社区
- 热度：Twitter热度/Google趋势
- 时间：上线1h/24h/7d

模型层:
- 每个 P小将 一个画像
- 新币来了 → 匹配度打分

执行层:
1. 新币监控
2. 画像匹配 → 预测买入
3. 提前买入
4. 监控 P小将 持仓变化（确认信号）
5. 卖出
```

## P小将 画像示例

| P小将 | 市值偏好 | 叙事偏好 | 买入时机 | 止盈 |
|-------|----------|----------|----------|------|
| @A | 5位 | meme | 上线即买 | 2x |
| @B | 6位 | 链游 | 回调买 | 5x |
| @C | 7位+ | AI | 稳定买 | 10x+ |

## 策略流程

```
1. 模型预测：A 会买 X
2. 我先买 X
3. A 买了（确认信号）
4. FOMO 来了，币涨
5. 我卖出
```

## 实现步骤

1. **数据收集** - 选定 P小将，获取历史买入+推文
2. **画像构建** - 打标签，建模型
3. **监控执行** - 新币监控 + 自动买入 + 卖出
